# Kessler Repository Information

## Summary
Kessler is a satellite tracking and orbital debris monitoring application providing a UNOOSA (United Nations Office for Outer Space Affairs) satellite registry viewer. The project combines a Python FastAPI backend that fetches and serves satellite orbital data with TLE (Two-Line Element) information from CelesTrak, and a React frontend for interactive visualization and filtering of satellite data.

## Repository Structure

### Main Components
- **api.py**: FastAPI backend application serving satellite data, TLE information, and orbital parameters
- **react-app/**: React frontend application with Vite build tool
- **Data Import Scripts**: Python utilities for importing and converting satellite registry data from various sources (UNOOSA, CelesTrak, Space-Track)
- **Data Files**: CSV registry files containing satellite metadata (UNOOSA registry, merged registries)
- **Jupyter Notebook**: kessler.ipynb for data exploration and analysis
- **un/**: Subdirectory containing UN registry conversion utilities

## Backend (Python FastAPI)

### Language & Runtime
**Language**: Python  
**Version**: Python 3.11  
**Runtime**: uvicorn (ASGI server)  
**Port**: 127.0.0.1:8000

### Main Dependencies
- **fastapi**: Web framework and API routing
- **pandas**: Data manipulation and CSV handling
- **numpy**: Numerical operations
- **requests**: HTTP client for fetching TLE data
- **BeautifulSoup (bs4)**: Web scraping
- **pdfplumber**: PDF parsing for document extraction
- **uvicorn**: ASGI application server

### Application Structure
- **Entry Point**: `api.py` - defines FastAPI app instance
- **API Endpoints**: Query-based endpoints for satellite data filtering and TLE lookups
- **TLE Caching**: In-memory cache with 1-hour TTL fetching from CelesTrak endpoints:
  - `stations.txt`, `resource.txt`, `sarsat.txt`, `dmc.txt`, `weather.txt`, `geo.txt`, `iss.txt`
- **Data Source**: `unoosa_registry.csv` loaded at startup
- **CORS Configuration**: Open CORS for all origins

### Build & Installation
```bash
# Install dependencies (requires pip)
pip install fastapi uvicorn pandas numpy requests beautifulsoup4 pdfplumber

# Run the API server
python -m uvicorn api:app --host 127.0.0.1 --port 8000
```

## Frontend (React + Vite)

### Language & Runtime
**Language**: JavaScript/JSX  
**Runtime**: Node.js  
**Framework**: React 19.2.3  
**Build Tool**: Vite 7.2.7  
**Port**: localhost:3000

### Main Dependencies
- **react**: UI framework (v19.2.3)
- **react-dom**: React DOM binding (v19.2.3)
- **vite**: Build tool and dev server (v7.2.7)
- **@vitejs/plugin-react**: Vite React plugin (v5.1.2)

### Application Structure
- **Entry Point**: `src/main.jsx`
- **Root Component**: `src/App.jsx`
- **Components**:
  - `Filters.jsx`: Filter UI for satellite search
  - `DataTable.jsx`: Data grid display

- **Styling**: CSS modules per component

### Build & Installation
```bash

cd react-app

# Install dependencies
npm install

# Development server
npm run dev

# Build for production
npm run build

# Preview production build
npm run preview
```

### Vite Configuration
- Dev server on port 3000
- API proxy: `/api` routes to `http://127.0.0.1:8000`
- React plugin enabled for JSX support

## Project Operations

### Starting Both Services
```bash
# From repository root
./start.sh
```

This script:
- Cleans up any processes on ports 8000 and 3000
- Starts FastAPI backend on http://127.0.0.1:8000
- Starts React dev server on http://localhost:3000
- Displays API documentation at http://127.0.0.1:8000/docs

### Data Files
- **unoosa_registry.csv** (1.19 MB): Main UNOOSA satellite registry loaded by API
- **unoosa_registry_import.csv**: Imported data before merge
- Various data generated by import scripts

### Utility Scripts
- `import_celestrak.py`: Import CelesTrak TLE data
- `import_spacetrack.py`: Import Space-Track data
- `import_unoosa_data.py`: Import UNOOSA registry data
- `supplement_celestrak.py`: Enhance CelesTrak data
- `scrape_orbital_params.py`: Extract orbital parameters from PDF documents
- `convert_un_registry.py`: Convert UN registry format
- `merge_registries.py`: Merge multiple registry sources

## Configuration
- **API CORS**: Fully open (allow_origins=["*"])
- **TLE Cache**: 1-hour TTL (3600 seconds)
- **Environment**: Loads data files from current directory
- **Jupyter**: kessler.ipynb for exploratory analysis

## Architecture Notes
- **Separation of Concerns**: Clear backend/frontend separation with API proxy
- **Data Layer**: Pandas-based in-memory data processing, CSV-backed persistence
- **Frontend**: Component-based React architecture with CSS modules
- **API**: RESTful FastAPI design with automatic OpenAPI documentation
